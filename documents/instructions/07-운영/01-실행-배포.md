# 실행 / 배포

> 키 없이도 로컬에서 전체 흐름이 동작한다(인메모리 + mock AI). 실 서비스는 Supabase·Google 키를 채운다. 지금 실제 배포는 하지 않으나 즉시 배포 가능하게 준비돼 있다(무상태·Docker).

## 로컬 실행
### 백엔드 (http://localhost:8000)
```bash
cd backend
python -m venv .venv
.venv/Scripts/python.exe -m pip install -e ".[dev]"   # Windows (mac/Linux는 source .venv/bin/activate)
cp .env.example .env
.venv/Scripts/python.exe -m uvicorn app.main:app --reload --reload-dir app --port 8000
.venv/Scripts/python.exe -m pytest                     # 테스트
```
> `--reload-dir app`로 app/만 감시(로그/임시파일이 reload storm을 일으키지 않게). 로그는 backend/ 밖에 둔다.

### 프론트엔드 (http://localhost:3000)
```bash
cd frontend
npm install
cp .env.example .env.local
npm run dev            # 개발 서버
npm run build && npm run lint   # 검증
```

## 동작 모드
| 모드 | 조건 | `/health` |
| --- | --- | --- |
| 개발(키 없음) | Supabase·Google 키 없음 + `DEV_AUTH=true` | `{storage:in-memory, ai:mock}` — 결정적 mock, dev 토큰 로그인 |
| 실키 | Supabase(service_role·JWT)·Google 키 + `DEV_AUTH=false` | `{storage:supabase, ai:google}` — 실 DB·실 AI, Google 로그인 |

## 환경 변수
**백엔드 `.env`**
```
SUPABASE_URL=                     # 비우면 인메모리
SUPABASE_ANON_KEY=
SUPABASE_SERVICE_ROLE_KEY=        # 서버 전용(노출 금지)
SUPABASE_JWT_SECRET=              # 있으면 dev 토큰 비활성(fail-closed)
GOOGLE_API_KEY=                   # 비우면 mock AI
GEMINI_MODEL_PRO|FLASH|FLASH_LITE, GEMINI_EMBED_MODEL, IMAGEN_MODEL
ADMIN_EMAILS=                     # 관리자 화이트리스트
ALLOWED_ORIGINS=http://localhost:3000   # CORS
DEV_AUTH=false                    # 운영 false
APP_ENV=dev|test|prod             # prod에서 Supabase 자격 없으면 기동 거부
```
**프론트 `.env.local`**
```
NEXT_PUBLIC_API_BASE_URL=http://localhost:8000
NEXT_PUBLIC_SUPABASE_URL=         # 비우면 개발 로그인 모드
NEXT_PUBLIC_SUPABASE_ANON_KEY=    # 공개용(anon)
```
> 시크릿(service_role·JWT·Google 키)은 절대 커밋하지 않는다. `.env`는 .gitignore 대상.

## 실 Google 로그인 활성화
1. 프론트 `.env.local`에 Supabase URL·anon 키.
2. Supabase Auth에서 Google provider 활성화 + Google Cloud OAuth 클라이언트(리디렉션 URI에 Supabase 콜백 등록).
3. 백엔드 `DEV_AUTH=false` + JWT 시크릿. 백엔드는 ES256 JWKS로 검증한다.

## 데이터베이스
Supabase(PostgreSQL + pgvector). 마이그레이션은 `backend/app/db/migrations/`에 순번으로 적용(테이블·RLS·HNSW 인덱스·RAG RPC). 삽화는 Storage `illustrations`(public) 버킷.

## 배포 준비 (지금 배포는 X)
- 백엔드: `Dockerfile`·`.dockerignore`·`/health`·`/health/ready`(레디니스 503). 컨테이너(Render/Fly/Railway). `ALLOWED_ORIGINS`에 배포 도메인.
- 프론트: Vercel(Next.js) — `NEXT_PUBLIC_*` 환경 변수.
- DB/Storage/Auth: Supabase(클라우드). 상세 절차 `backend/DEPLOY.md`.
- 무상태(인메모리/로컬파일 의존 제거)라 수평 확장 가능.

## 알림 상시 가동
관리자 알림은 백엔드가 상시 가동돼야 하며 주기는 `app_settings.notify_interval_sec`(기본 180초, 관리자가 변경)로 동작한다.
