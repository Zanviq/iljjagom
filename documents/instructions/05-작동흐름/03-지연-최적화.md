# 지연 최적화

> AI 생성은 본질적으로 시간이 걸린다. 일짜곰은 무거운 작업을 **학생이 다른 일을 하는 동안 백그라운드로 미리** 만들어 둠으로써 체감 대기를 없앤다.

## 원칙
1. **백그라운드 선생성(prefetch)**: 다음에 필요할 산출물(설계·다음 챕터·전·결·삽화·학습 교재)을 미리 초안 생성해 저장한다. 학생이 변경하면 해당 부분만 갱신.
2. **저장본 우선**: 이미 생성된 챕터는 재생성 없이 저장본을 즉시 스트리밍.
3. **단일성 락**: 한 책에서 여러 prefetch가 동시에 돌아도 중복 생성하지 않도록 `(book_id, idx)` 락을 공유(`services/prefetch.py`).

## 어디서 미리 만드나
| 구간 | 트리거 | 효과 |
| --- | --- | --- |
| 설계(Bible) + 첫 챕터 | 기획 대화가 충분(readyToWrite) | "이야기 시작" 클릭 시 즉시 진입 |
| 다음 장(본문+삽화) | 현재 장 `done` | "다음 장" 진입 시 대기 0 |
| **전·결 전체** | **기·승 완료** | 중간활동 푸는 동안 전·결 생성(`prefetch_arc`) |
| **중간 퀴즈** | **기·승 완료** | 중간활동 진입 시 실 AI 퀴즈 대기 0(`midactivity.prefetch_mid_quiz`) |
| **마무리 학습 교재·퀴즈** | **결(마지막 장) 스트림 종료** | "학습하러 가기" 진입 시 실 AI 퀴즈·교재 대기 0(`learning.prefetch_learning`) |
| 낱말 뜻 | 본문 확정 시 낱말 선정과 함께 | 단어 탭 시 즉시 |

## 단계 구조 — 기승 → 중간활동 → 전결
전·결은 학생 선택(기·승 내용)에 종속되어 미리 만들기 어렵다. 그래서 기·승이 끝나면 **중간활동(중간 퀴즈/독후감, 필수)**을 끼우고, 학생이 그걸 푸는 동안 전·결을 백그라운드로 생성한다. 진입 시점엔 이미 저장돼 있어 즉시 스트리밍된다.
```
[기·승 협업] → [중간활동(필수)] → [전·결] → … → [결] → [마무리 학습]
   └ 기·승 완료 순간 prefetch_arc(전·결 챕터) + prefetch_mid_quiz(중간 퀴즈) 선생성
                                        결 스트림 종료 순간 prefetch_learning(마무리 교재·퀴즈) 선생성 ┘
```

## 캐싱
- 학습 교재는 `learning_artifacts`에 `type="learning_set"`으로 1회 저장·재사용(본문 시그니처 `idx:char_count`가 키, 본문 변경 시 자동 무효화·재생성). 중간활동은 `type="mid_activity"`로 분리.
- 캐시 키(`sourceSig`)에는 **스키마 버전(`v2`)**이 포함된다. 생성 로직이 바뀌면(예: 고정 템플릿 → 본문 기반·학년 맞춤 실 AI 퀴즈) 버전을 올려 옛 템플릿 캐시를 자동 무효화한다.
- 퀴즈는 셔플된 정답 위치를 캐시에 1회 고정 저장(재방문 동일, "정답이 항상 1번" 문제 해소). 감정 곡선·편지는 학생 산출물이라 캐시 대상이 아니라 `learning_results`에 저장.

## 읽기 경로 쿼리 일괄화 (N+1 제거)
자주 호출되는 읽기 경로(`list_books`·`get_book_detail`·`get_mid_activity`)는 문단을 책 단위로 **1회 일괄 조회**(`store.list_paragraphs_for_book`)하고, Bible·챕터·문단 수를 호출 체인에 주입한다. 챕터마다 따로 묻던 중복 Supabase 왕복을 없애 목록·상세 로딩을 빠르게 한다.

## 스트림 견고화
지연이 길어 SSE가 끊기던 문제는 (a) 선생성으로 생성 지연 자체를 줄이고, (b) 백엔드 하트비트·`retryAfter`·`done` 미수신 재시도, (c) `?from=` 정밀 재개(UTF-16 오프셋)로 해결한다(→ `04-API/03-SSE-실시간.md`).
