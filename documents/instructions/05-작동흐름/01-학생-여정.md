# 학생 여정 (end-to-end)

> 로그인부터 발표까지, 책 한 권이 만들어지는 전 과정. 각 단계의 화면·API·AI를 함께 표시한다.

```
[로그인]  Google OAuth → /auth/callback → (최초)온보딩
   │       온보딩: 역할=학생 + 학급 코드 + 보호자 동의
   ▼
[홈 /home]  GET /me(classId)·GET /classes/{id}/prompts·GET /books
   │        · 발제 카드에서 "이 주제로 새 책" → POST /books (status=planning)
   │        · "이어 읽던 이야기" 카드로 재진입
   ▼
[기획 /plan]  POST /books/{id}/plan/messages  (대화 AI, Flash-Lite)
   │          인물·배경 인터뷰 → readyToWrite/interviewClosed 되면 협업으로
   │          ※ 이 동안 백그라운드로 Bible 설계 선생성(POST /design 즉시화)
   ▼
[기·승 협업 /write]  POST /books/{id}/chapters/{n}/collab  (free 챕터)
   │   좌: 본문 문단 누적 / 우: AI 대화. 한 마디 = 한 문단. 흐름/주제 이탈 시 지도.
   │   문단이 충분 → chapterComplete
   ▼
[중간활동 (필수)]  GET /books/{id}/mid-activity → POST .../complete
   │   중간 퀴즈/독후감을 푸는 동안, 백그라운드로 전·결(guided) 챕터 선생성(prefetch_arc)
   ▼
[전·결 독서 /read]  GET /books/{id}/chapters/{n}/stream  (집필 AI, SSE)
   │   유도모드: 삽화 선노출 + 능동 질문 → 탭하면 본문 글자 단위. 단어 도움(GET /words).
   │   선생성 덕에 진입 즉시 스트리밍. 완독 → 완독 이벤트(POST /events)
   ▼
[학습 /learn]  GET /books/{id}/learning  (학습 교재, 캐시)
   │   낱말 카드 · 퀴즈(정답 무작위) · 감정 곡선(학생 입력) · 인물 편지(인물 선택, POST /letters)
   │   · 독후감. 결과는 POST /books/{id}/learning-results 저장.
   ▼
[발표]  POST /books/{id}/board-posts → 교사 승인 → 학급 게시판 공개
```

## 단계별 책 상태
`planning`(기획) → `writing`(기·승 협업·중간활동·전·결) → `done`(완독·학습 가능·발표 가능).

## 곰 작가(총괄 AI)
홈 등 메인 화면에서 우하단 곰 작가 버튼으로 도움을 청한다. "책 쓰고 싶어" → 발제 안내 + "책 만들러 가기" 버튼(자동 책 생성 후 기획 이동). 도서 제작/독서 화면(`/plan`·`/write`·`/read`)에는 버튼이 없다(몰입 보호).

## 지연이 느껴지지 않는 이유
무거운 생성(설계·다음 장·전·결·삽화·학습 교재)은 학생이 다른 걸 하는 동안 **백그라운드로 미리** 만들어 둔다(→ `03-지연-최적화.md`).
