# 03 · 용어집

> 일짜곰 문서 전반에서 쓰이는 핵심 용어를 한곳에 정의한다. 도메인 개념부터 AI 파이프라인, 데이터·보안 용어까지 묶었다.

---

## 도메인 · 콘텐츠

**Bible (바이블)**
책 한 권의 모든 정보를 담은 단일 진실 원천(single source of truth). 인물 카드, 세계관, 사건 요약, 교사 학습 목표, 후반부의 큰 줄기를 구조화된 JSON으로 저장한다. 모든 AI 역할이 Bible을 참조하므로, 챕터가 늘어도 인물 성격·말투·삽화 외형의 일관성이 유지된다. 책당 한 번 설계 AI가 생성한다.

**발제 (prompt)**
교사가 학급에 내는 학습 과제. 주제, 학습 목표(여러 개), 평가 방식(없음/퀴즈/독후감), 언어(한국어/영어)로 구성된다. 같은 발제 아래 학급 학생들이 각자 다른 이야기를 만들면서도 동일한 학습 목표를 거친다. (코드·DB에서는 `prompts`)

**free / guided 모드**
챕터의 집필 방식 구분.
- **free(자유)**: 기·승 단계. 학생이 AI와 대화하며 문단 단위로 함께 짓는다.
- **guided(유도)**: 전·결 단계. AI가 결말을 비밀로 한 챕터씩 펼치고 학생은 첫 독자로 읽는다.

설계 시 챕터의 앞 절반이 free, 뒤 절반이 guided로 분배된다.

**중간 활동 (mid activity)**
기·승(자유 집필)이 끝난 뒤 전·결(유도 집필)로 넘어가기 전에 **반드시** 거치는 학습 단계. 중간 퀴즈·중간 독후감으로 구성되며, 기·승 범위까지의 내용을 다룬다. 학생이 이를 푸는 동안 전·결 챕터가 백그라운드에서 선생성된다.

**secretArc (비밀 전개)**
전·결의 결말과 후반 큰 줄기. Bible에 담기지만 기·승 협업 단계에서는 절대 노출되지 않으며, 유도 집필이 진행되며 마지막 순간에 회수된다.

**학급 게시판 (class posts)**
완성한 책을 학급에 발표·소개·공유하는 공간. 학생이 발표글을 올리면 기본적으로 교사 승인 후 학급 멤버에게 공개된다. (코드·DB에서는 `class_posts`)

## 협업 집필

**협업 (collab)**
자유 집필 단계에서 학생과 AI가 한 문단씩 번갈아 이야기를 짓는 방식. 좌측 본문, 우측 대화방의 2단 화면에서 이루어진다. 학생 한 마디가 한 문단 생성으로 이어진다.

**문단 (paragraph)**
협업의 최소 단위. 학생의 한 응답이 한 문단을 만든다. 통째 챕터 생성과 대비된다. 누적된 문단을 순서대로 이으면 챕터 본문이 된다. (코드·DB에서는 `chapter_paragraphs`, 협업 대화 턴은 `writing_turns`)

**지도 (coaching)**
협업 중 학생의 의도가 학습 주제에서 벗어나거나 직전 문단과 흐름이 어색할 때, AI가 무작정 따르지 않고 더 나은 방향을 제안하는 응답. "학생 의도 긍정 → 대안 제시 → 근거 설명" 형식을 따른다. 강제가 아니라 제안이라, 학생이 고수하면 그 의도대로 생성된다(아동 주도성 존중).

## AI 파이프라인

**총괄 AI (overseer · 곰 작가)**
사용자에게 보이는 이야기 작가 페르소나이자, 여러 AI 역할의 동작을 조율·합성해 하나의 일관된 응답으로 전달하는 총괄 계층. 내부 데이터를 합성해 아이에게 말을 건다.

**설계 AI / 집필 AI / 편집 AI / 대화 AI**
4계층 AI 파이프라인의 역할. 호출 빈도와 필요 품질에 맞춰 모델 등급이 매칭된다.
- **설계 AI**: 책당 1회. Bible 생성, 학습 목표를 챕터에 분배.
- **집필 AI**: 챕터마다. 본문 스트리밍 생성, RAG 인출.
- **편집 AI**: 챕터 직후 비동기. 말투·시간선·어휘·학습 목표 도달도 검수, 미달 단락 재생성.
- **대화 AI**: 가장 빈번. 튜터·인물 페르소나·유도 모드 능동 대화·안전 게이트.

**이미지 AI**
챕터마다 삽화 한 장을 생성한다. Bible의 인물 카드를 입력받아 삽화 외형의 일관성을 유지한다.

**Skill (스킬)**
AI가 작업을 수행할 때 호출하는 도구 단위. `ask_user`(질문 후 학생 답 대기), `generate_text`(본문 생성), `retrieve_context`(RAG 인출), `update_bible`(Bible 보강), `check_safety`(안전 점검), `summarize`(요약) 등이 있다.

**ReAct (리액트)**
AI가 thought(사고) → skill(도구 호출) → observation(관찰) 루프를 반복하며 작업을 진행하는 추론 방식. `ask_user` 액션이 세션을 일시정지(awaiting_user)했다가 학생 답으로 재개하는 메커니즘이 협업 집필의 토대가 된다.

**트레이스 (trace · ai_sessions / ai_steps)**
모든 AI 동작의 실행 기록. `ai_sessions`는 한 작업 세션(어떤 역할이 무슨 작업을 했는지), `ai_steps`는 그 안의 개별 단계(스텝별 입력 프롬프트 스냅샷·출력 포함)를 남긴다. 관리자 관측과 교사 열람의 데이터 원천이다.

## 데이터 · 검색 · 보안

**RAG (Retrieval-Augmented Generation)**
생성 시 Bible에서 필요한 항목만 인출해 함께 넣는 방식. 챕터가 늘어도 호출당 토큰이 일정하게 유지되어 비용이 안정되고, 장편에서도 내부 일관성이 보장된다.

**pgvector**
PostgreSQL의 벡터 검색 확장. 임베딩 벡터(gemini-embedding-001, 768차원)를 저장하고, HNSW 인덱스로 코사인 유사도 근사 최근접 검색(ANN)을 SQL 안에서 수행한다. 별도 벡터 DB 없이 RAG를 구현한다.

**안전 강도 (safety level)**
AI 본문·응답의 콘텐츠 안전 수준. `relaxed | standard | strict` 세 단계(enum)로 정의되며 기본값은 `standard`다. 해석 체인은 **발제 > 학급 설정 > 전역 설정 > 코드 기본** 순으로, 교사가 학급별로 오버라이드할 수 있다.

**RLS (Row Level Security · 행 수준 보안)**
PostgreSQL의 행 단위 접근 제어. 학급·학교 단위로 데이터를 격리해 "교사는 자기 학급 책만 열람", "학생은 자기 책만 접근" 같은 규칙을 DB 차원에서 강제한다. 모든 테이블에 적용된다.

**SSE (Server-Sent Events)**
서버가 클라이언트로 데이터를 실시간 단방향 전송하는 방식. 본문을 글자 단위로 스트리밍하며, 자동 재연결과 하트비트로 생성 지연 중에도 연결을 안정적으로 유지한다.

**선생성 (prefetch)**
다음에 필요한 작업(Bible·다음 챕터·전결·학습 교재 등)을 학생이 다른 활동을 하는 동안 백그라운드에서 미리 생성·저장하는 것. 공통 락으로 한 책에서 중복 생성을 방지하며, 진입 시 생성 대기 없이 즉시 제공한다.
