# 관리자 기능 — Control Room

> 관리자 환경은 딥 네이비·시안/라임 신호·mono 수치의 운영/관측 콘솔(`.theme-admin`). 서비스가 정상 작동하는지 **보고 테스트**하는 것이 목적이라 강한 권한·노출을 허용하되, 관리자 전용 + 감사(audit) 기록된다.

## 접근 제어
`profiles.role='admin'` + `ADMIN_EMAILS` 화이트리스트 둘 다 통과해야 진입. 모든 `/admin/*` 엔드포인트는 역할 검증 + `audit_log` 기록.

## 콘솔 9탭
| 탭 | 내용 |
| --- | --- |
| **개요** | 실시간 상태판: 동작 중 세션·오류·오늘 토큰/비용·미처리 안전신호·사용자/책/챕터 수. **각 지표 박스를 클릭하면 그 기록 목록으로 드릴다운.** |
| **AI 세션** | 세션 목록(role·model·status·진행 스텝·토큰). running/awaiting_user 실시간 표시. |
| **사용자** | 사용자 표(역할·학급·상태). 역할 변경·학급 배정·비활성(마지막 관리자 보호). |
| **대화** | 메시지 검색·필터(사용자·책·kind·기간). |
| **토큰·비용** | `token_usage` 집계(모델별·역할별·기간), 추세. |
| **안전** | safety_flags·held 편지 전역 뷰·종결. |
| **설정** | 역할별 Gemini 모델·런타임 수치(알림주기·안전강도 등)를 **선택형**으로 편집. 시크릿은 존재 여부만 표시(값 비노출). |
| **알림** | 발송(전체/역할/사용자)·수신함. 백엔드 상시 가동 + 주기(`notify_interval_sec`, 기본 180초) 자동 알림. |
| **백업** | 데이터 내보내기/복원(merge/overwrite). |

## 관측 재설계 — 드릴다운 + SessionInspector
관리자의 핵심 가치는 **"AI가 왜 이렇게 했는지"를 끝까지 추적**하는 것이다.

- **정보 계층 드릴다운**: 사용자 → 책 → 단계(기획/설계/집필/편집/대화/학습) → 세션 → 스텝 → 메시지. 개요 박스·목록·사용자 항목을 클릭해 점점 깊이 들어간다. 대화 페이지는 "전체 목록 + 그 아래 사용자 목록 → 사용자 클릭 시 그 사용자의 대화/책 기록"으로 묶여 보인다.
- **SessionInspector(우측 사이드바)**: 세션·대화 기록을 클릭하면 우측 패널이 열려 **그 세션에 들어간 시스템/유저 프롬프트(스냅샷)·사용한 스킬·관측·토큰·대화 전문·이미지 생성 과정**을 한 번에 보여준다.
- 관련 API: `GET /admin/users/{id}/overview`, `GET /admin/books/{id}/timeline`, `GET /ai/sessions/{id}`(transcript·프롬프트 동봉).

이 관측 체계가 있으면 "총괄 AI가 같은 말만 반복" 같은 문제를 어떤 프롬프트·스킬·결정 때문인지 즉시 진단할 수 있다.

## 설정은 선택형
설정 값은 자유 입력이 아니라 **드롭다운/토글/스텝퍼**로 고른다. 키별 허용값 스키마(enum)를 단일 출처로 두고 백엔드가 enum·범위를 검증한다(예: 모델 = 2.5-pro/flash/flash-lite, 안전강도 = relaxed/standard/strict).

## 주의
콘솔이 강력하므로 관리자 게이트·감사·시크릿 비노출이 필수. 대화/세션 열람은 미성년 데이터이므로 테스트·관측 목적임을 전제로 한다.
