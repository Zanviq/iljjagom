# 01 · 학생 기능

> 학생은 일짜곰의 주 사용자다. 발제를 고르는 순간부터 책을 만들고, 함께 쓰고, 읽고, 학습 활동으로 마무리해 학급에 발표하기까지 한 흐름으로 이어 간다. 이 문서는 학생이 만나는 모든 화면의 목적·구성요소·할 수 있는 행동을 정리한다.

---

## 0. 학생 흐름 한눈에

```
홈(내 책장) → 기획(인터뷰) → 기·승 협업 집필 → 중간 활동(필수) → 전·결 독서 → 학습 활동 → 발표
```

- **홈**에서 발제를 골라 새 책을 만들거나, 만들던 책을 이어간다.
- **기획**에서 AI와 대화하며 주인공·배경을 정한다. 준비가 끝나면 협업 집필로 넘어간다.
- **기·승**은 학생이 AI와 한 문단씩 함께 짓는 자유 집필이다(좌 본문 / 우 대화).
- **중간 활동**(퀴즈·독후감)을 반드시 끝내야 전·결로 넘어간다. 그동안 전·결이 미리 만들어진다.
- **전·결**은 AI가 결말을 비밀로 펼치는 유도 독서다.
- **학습 활동**으로 낱말·퀴즈·감정 곡선·편지·독후감을 하고, 마지막에 학급에 **발표**한다.
- 모든 화면 우측 아래에는 무엇이든 물어볼 수 있는 **총괄 AI(곰 작가)** 사이드바가 떠 있다.

학생 화면은 따뜻한 2인칭 말투, 큰 글자, 한 화면 한 행동을 원칙으로 한다.

---

## 1. 홈 — 내 책장

학생의 메인 허브. 새 이야기를 시작하고 만들던 책을 이어가는 곳이다.

### 목적
- 만들던 책을 빠르게 이어가게 한다.
- 교사가 낸 발제로 새 책을 시작하게 한다.

### 구성요소

| 영역 | 내용 |
| --- | --- |
| 인사 헤더 | 학급명과 이름을 부르는 환영 문구("안녕, ○○아!"). |
| 이어 읽던 이야기 | 내 책 목록. 각 책 카드에 제목·표지, 상태 배지(기획 중 / 읽는 중 / 다 읽음), 진행률 바("n / m장"), 이어가기 버튼. |
| 새 이야기 시작하기 | 교사가 낸 발제 카드 목록. 각 카드에 주제와 학습 목표 태그, "이 주제로 새 책 만들기" 버튼. |
| 후속 상태 배지 | 학습을 마친 책에는 발표 상태(발표 대기 / 공개됨)가 함께 표시된다. |

### 할 수 있는 행동
- 발제를 골라 새 책을 만들면 곧바로 기획 화면으로 이동한다.
- 만들던 책의 상태에 따라 "기획 이어가기 / 이어 읽기 / 다시 읽기"로 진입한다.
- 학급에 가입하지 않은 학생에게는 학급 코드 안내만 보이고, 나머지 섹션은 숨겨진다.
- 아직 발제가 없으면 "조금만 기다려 주세요" 안내가 보인다.

---

## 2. 기획 — 이야기 만들기 (인터뷰)

AI 인터뷰어와 대화하며 주인공·배경·분위기를 정하는 자유 집필의 도입부다.

### 목적
- 대화만으로 이야기의 인물과 배경을 정한다.
- 충분히 정해지면 협업 집필로 전환한다.

### 구성요소

| 영역 | 내용 |
| --- | --- |
| 대화 패널 | AI와 학생의 말풍선이 오가는 채팅. AI가 먼저 질문을 던진다. 학생 말풍선은 우측, AI는 좌측. |
| 입력창 | 줄바꿈이 가능한 입력창과 보내기 버튼. 입력이 비면 보내기는 비활성. |
| 인물 카드 | 대화에서 쌓인 주인공 이름·성격 특성이 미리보기로 표시된다. 정해지기 전에는 "이름은 아직 비밀". |
| 시작 버튼 | "이야기 시작하기". 준비가 끝나기 전에는 비활성이다. |

### 인터뷰 종료와 전환
- 학생이 인물·배경을 충분히 말하면 **인터뷰가 마무리 모드로 닫힌다.** 이때부터 AI는 새 질문을 멈추고 공감·칭찬만 한다("와, 개성 넘치는 친구들이네요!").
- 마무리 모드에서도 학생이 더 들려주고 싶은 내용을 말하면 그 내용은 설계에 반영된다.
- "이야기 시작하기"를 누르면 AI가 책 한 권의 설계(인물·세계관·사건·학습 목표)를 만든다. 이 설계는 학생이 기획 중일 때 **백그라운드에서 미리 준비**되므로, 버튼을 눌렀을 때 기다림이 짧다.
- 설계가 끝나면 기·승 협업 집필 화면으로 이동한다.

### 할 수 있는 행동
- 대화를 주고받으며 인물·배경을 만든다.
- 준비가 끝나면 이야기를 시작해 협업 집필로 넘어간다.

---

## 3. 기·승 협업 집필 (자유 집필)

기·승 단계는 AI가 통째로 써 주는 글을 읽는 것이 아니라, **학생과 AI가 한 문단씩 함께 짓는 협업**이다. 화면은 좌(본문) / 우(대화) 2단이다.

### 목적
- 학생의 의도가 본문에 곧바로 반영되는 집필 경험을 준다.
- 흐름·주제에서 벗어나면 AI가 지도해 더 나은 방향을 제안한다.

### 구성요소

| 영역 | 내용 |
| --- | --- |
| 상단 표시 | "현재 장 / 전체 장" 진행 표시. |
| 좌 — 본문 | 협업으로 쌓인 문단이 순서대로 누적되는 본문 카드. 새 문단이 부드럽게 추가된다. |
| 우 — 대화방 | AI의 진행 질문·지도와 학생의 입력이 오가는 채팅. 입력창 포함. |
| 지도 선택 버튼 | AI가 지도(제안)를 했을 때 "이대로 갈래요 / 제안대로 할래요" 두 버튼이 말풍선에 붙는다. |

### 상호작용 모델 (한 문단 = 한 협업 턴)
1. AI(곰 작가)의 여는 말은 고정 문구가 아니라 그 책의 인물 이름·장 번호로 만들어진다. 첫 장이면 "○○의 이야기, 어떻게 시작할까?", 2장 이상이면 "지금까지 이어온 ○○의 이야기, N장에서는 무슨 일이 생길까?"처럼 장 연속성을 담는다.
2. 학생이 한 마디 하면 그 의도로 **한 문단**이 좌측 본문에 생성된다.
3. AI는 문단마다 끝에 "이제 어떻게 진행돼?"류 **진행 질문 한 개**를 던진다.
4. 학생의 의도가 (a) 학습 주제에서 벗어나거나 (b) 직전 문단과 흐름이 어색하면, AI는 그냥 따르지 않고 **지도**한다.
   - 먼저 긍정하고("물론 그것도 좋지!"), 대안을 제시하고, 근거를 댄다("앞 문단과 더 잘 이어지고, 수업 주제도 잘 드러나거든").
   - 학생이 받아들이면 대안으로, 고수하면 학생 의도대로 생성한다(아동의 주도성을 존중).
5. 문단이 충분히 쌓이면 AI가 "기·승이 멋지게 완성됐어! 잠깐 퀴즈 풀고 올래?"라며 중간 활동으로 안내한다.

### 행동·규칙
- 한 응답 = 한 문단. 통째 생성은 없다.
- AI가 얼마나 지도(간섭)할지는 교사가 학급별로 정한 **AI 지도 강도**(끔/흐름만/흐름+주제)에 따른다. 어느 강도에서도 학생이 고수하면 학생 의도대로 생성된다.
- 결말은 기·승 협업에서 절대 드러나지 않는다.
- 새로고침 후에도 좌(문단)·우(대화)가 그대로 복원된다.
- 협업으로 완성된 기·승 챕터는 독서 화면에서 그대로 읽힌다(읽기 경험은 변하지 않는다).

---

## 4. 중간 활동 (필수)

기·승이 끝나면 전·결로 바로 가지 않고 **중간 퀴즈·중간 독후감**을 반드시 거친다.

### 목적
- 기·승까지의 내용을 점검하며 학습 목표를 다진다.
- 학생이 활동하는 동안 전·결 챕터를 백그라운드에서 미리 만들어, 다음 단계 진입을 빠르게 한다.

### 구성요소·행동
- 기·승 범위까지의 **실제 본문을 읽혀** 만든 **중간 퀴즈**(객관식, 내용 이해 + 학습 목표 적용)와 **중간 독후감**(질문 + 빈칸)이 제시된다. 퀴즈 문항은 학년 수준(1~2 / 3~4 / 5~6학년)에 맞춰 난이도가 조절된다.
- 이 활동은 **건너뛸 수 없다.** 완료해야 전·결 독서로 진행한다.
- 학생이 활동을 푸는 동안 전·결 챕터(삽화 포함)와 중간 퀴즈가 미리 생성·저장되므로, 진입 시 새 생성 대기가 거의 없다.

---

## 5. 독서 — 전·결 (유도 집필)

전·결은 AI가 결말을 비밀로 두고 한 챕터씩 펼치는 유도 독서다. 핵심 몰입 화면이다.

### 목적
- 결말이 궁금해 끝까지 읽게 한다.
- 읽으며 모르는 낱말을 익힌다.

### 구성요소

| 영역 | 내용 |
| --- | --- |
| 상단 | 책 제목과 "n / m장" 배지. |
| 삽화 | 챕터 그림. 유도 모드에서는 **본문보다 먼저** 삽화가 나오고 "이건 어떤 장면일까?" 같은 AI의 질문이 붙는다. |
| 유도 게이트 | 본문 공개 전, "그림을 보고 어떤 이야기일지 상상해 봐요" 카드 + "이야기 읽어볼까요?" 버튼. |
| 본문 | 한 글자씩 흘러나오는(스트리밍) 이야기 글. 다 나오면 멈춘다. 머리말·마크다운 흔적 없이 정제되어 나온다. |
| 단어 도움 | 본문에서 낱말을 선택하면 발음·뜻 팝업이 뜬다. |
| 이 장의 낱말 | 챕터 끝에 핵심 낱말 모음. 누르면 뜻이 보인다. 낱말은 학년 수준에 맞춰 선정된다. |
| 다음 장 / 마무리 | "다음 장으로" 버튼, 마지막 장이면 "다 읽었어요" 축하. |
| 학습 활동 진입 | "학습 활동 하러 가기" 버튼. |

### 할 수 있는 행동
- 삽화를 보고 본문을 연다.
- 본문을 읽고, 모르는 낱말을 선택해 뜻을 본다.
- 다음 장으로 넘어간다. 읽는 동안 다음 장이 미리 준비되어 대기가 짧다.
- 다 읽으면 학습 활동으로 이동한다.

> 참고: 연결이 잠시 끊겨도 자동으로 재시도·복원되며, 본문은 멈추지 않고 이어진다.

---

## 6. 학습 활동

읽은 이야기로 어휘·이해·표현 활동을 한다. 이야기 내용을 바탕으로 자동 생성된다.

### 목적
- 읽은 이야기로 배우고 자기 생각을 표현한다.

### 구성요소·행동

| 활동 | 내용 | 학생이 하는 일 |
| --- | --- | --- |
| 낱말 카드 | 단어·발음·뜻 카드 모음. | 낱말의 뜻을 익힌다. |
| 퀴즈 | **실제 이야기 본문을 바탕으로** 만든 객관식 문제(내용 이해 + 학습 목표 적용, 학년 수준에 맞춘 난이도). **정답 위치는 문항마다 무작위로 흩어진다**(같은 책은 같은 위치로 고정). | 보기를 골라 정답/오답 피드백을 받고 다시 푼다. |
| 독후감 채우기 | 질문 + 힌트 + 답 적기. | 빈칸에 자기 생각을 적는다. |
| 감정 곡선 | **학생이 직접 입력**하는 활동. 장마다 감정 라벨을 고르고 강도를 조절하면 곡선이 실시간으로 그려진다. | 장별 감정을 골라 자기만의 곡선을 만들고 저장한다. |
| 인물에게 편지 쓰기 | 받는 인물을 **목록에서 선택**(타이핑하지 않음)하고 편지를 쓴다. | 인물을 고르고 편지를 보낸다. AI 인물이 답장한다. |

### 세부 규칙
- **감정 곡선**: 시스템이 임의로 그려 주지 않는다. 학생이 장마다 감정을 직접 골라 곡선을 만들고, 저장하면 다시 들어와도 복원된다. 미입력 장은 점선·회색으로 구분된다.
- **인물 편지**: 이야기에 등장하는 인물이 선택지로 제공된다. 인물이 한 명이면 자동 선택되고, 아직 인물이 없으면 편지 블록 대신 안내가 보인다. 위험한 내용이 감지되면 답장이 보류되고 "선생님이 확인한 뒤 답장을 받을 수 있어요" 안내가 뜬다.
- **인물 답장**: AI 인물은 자기 **종류·생김새**, 이야기의 **세계관·제목**, 그리고 학생과 **함께 겪은 본문 맥락**을 떠올려 그 인물답게 답장한다(추억을 구체적으로 곁들임). 단, 이야기의 결말은 끝까지 비공개로 둔다.
- 학습 활동은 한 번 만들어지면 캐시되어 다시 들어와도 빠르게 뜬다.

---

## 7. 학습 마무리와 발표

학습 활동 화면 맨 아래에는 끝맺음 섹션이 있다. 학생은 여기서 학습을 매듭짓고 후속 행동을 고른다.

### 구성요소

| 요소 | 내용 |
| --- | --- |
| 완료 요약 | 푼 퀴즈·쓴 독후감·그린 감정 곡선·보낸 편지 여부를 체크로 보여 준다. 각 활동을 저장하는 즉시(이벤트 기반) 체크 표시가 갱신된다. |
| 메인으로 돌아가기 | 학습 완료를 기록하고 홈으로 이동한다. |
| 학급에 발표하기 | 완성한 책을 학급 게시판에 발표 글로 등록한다. 책을 다 읽은(완독) 경우에만 활성. |

### 학급 게시판 (발표)
- "학급에 발표하기"를 누르면 책 요약과 학습 결과(감정 곡선·편지·퀴즈 점수 등)를 담은 발표 글이 만들어진다.
- 발표 글은 보통 **교사 승인 후** 학급에 공개된다(학급 설정에 따라 자동 공개일 수도 있다).
- 학급 게시판 화면에서는 학급 친구들의 공개된 발표 글을 표지·제목·작성자와 함께 볼 수 있고, 글을 눌러 책 요약을 읽을 수 있다.
- 반려된 글은 사유와 함께 다시 제출할 수 있다.
- 홈의 책 카드에는 "발표 대기 / 공개됨" 배지로 후속 상태가 표시된다.

## 8. 총괄 AI — 곰 작가 사이드바

학생 화면(메인)에는 무엇이든 물어볼 수 있는 총괄 AI "곰 작가"가 항상 함께한다.

### 목적
- 학생이 길을 잃지 않도록 돕고, "지금까지 내가 한 것"을 알려 주며, 다음 행동으로 안내한다.

### 구성요소

| 요소 | 내용 |
| --- | --- |
| 띄우기 버튼(FAB) | 화면 우측 아래 고정. "곰 작가에게 물어보기". |
| 사이드 드로어 | 우측에서 슬라이드로 열리는 대화 패널. 곰 작가 마크 + "무엇이든 물어봐요" + 대화 + 입력창. |

### 할 수 있는 행동
- 무엇이든 묻는다. 곰 작가는 **실제 데이터로 답한다.** "지금까지 내가 한 거 알려줘"라고 하면 만든 책 수·제목·진행 장수처럼 실제 수치를 담아 알려 준다(예고만 반복하지 않는다).
- 곰 작가의 답에는 이모지·마크다운이 섞이지 않고, 따뜻하고 쉬운 평문으로 나온다.
- 곰 작가는 학생이 책을 만들거나 페이지를 이동하도록 도울 수 있다.

> 곰 작가 사이드바는 학생 화면에서만 보인다. 교사·관리자 화면에는 노출되지 않는다.
