# 데이터 모델

> 모든 데이터는 Supabase(PostgreSQL)에 저장하며 **전 테이블 RLS** 적용. 벡터는 pgvector, 파일은 Supabase Storage. 마이그레이션은 `backend/app/db/migrations/`에 순번(0001~)으로 누적된다.

## 도메인 테이블
| 테이블 | 핵심 컬럼 | 비고 |
| --- | --- | --- |
| `profiles` | id(=auth.users), email, role(`student|teacher|admin`), grade, guardian_consent, display_name, status | 사용자 |
| `schools` · `classrooms` · `enrollments` | classrooms: school_id, teacher_id, name, **code**(가입코드), board_auto_publish / enrollments: (classroom_id, student_id) | 학급·등록 |
| `prompts` | classroom_id, topic, learning_objectives(jsonb[]), assessment(jsonb), language, status | 교사 발제 |
| `books` | student_id, classroom_id, prompt_id, status(`planning|writing|done`), title, total_chapters_planned, updated_at | 책 |
| `bibles` | book_id(PK), data(jsonb) | 단일 진실 원천 |
| `chapters` | book_id, idx, mode(`free|guided`), body(text), words(jsonb), char_count, review_status, illustration_path, paragraphCount(파생) | 챕터 |
| `chapter_paragraphs` | chapter_id, book_id, seq, body, source(`collab|ai|revise`) | 기·승 협업 문단 |
| `writing_turns` | chapter_id, book_id, role(`student|writer`), kind(`message|question|coaching`), content, paragraph_id | 협업 대화 턴 |
| `chapter_chunks` | book_id, chapter_id, content, embedding `vector(768)` | RAG (HNSW cosine) |
| `plan_messages` · `messages` | 기획 인터뷰 / 통합 대화(kind: overseer 등) | 대화 로그 |
| `learning_artifacts` | book_id, chapter_id, type(`vocab|quiz|essay|emotion|learning_set|mid_activity`), data(jsonb) | 학습 교재·캐시 |
| `learning_results` | book_id, student_id, type(`quiz|essay|emotion|letter`), data(jsonb) | 학생 학습 결과 |
| `letters` | book_id, student_id, to, body, status(`pending|answered|held|approved|rejected`), reply, reviewed_by/at, note | 인물 편지(승인 루프) |
| `events` | book_id, student_id, type, payload(jsonb) | 행동 로그(완독·재방문·단어터치 등) |
| `safety_flags` | book_id, student_id, source, category, severity, reason, status(`open|reviewed|resolved`), reviewed_by/at, letter_id | 안전 신호 |
| `class_posts` | classroom_id, book_id, student_id, title, intro, snapshot(jsonb), status(`pending|published|rejected`), reviewed_by/at | 학급 게시판(발표) |
| `class_settings` | classroom_id, settings(jsonb: 안전강도·기능 토글) | 학급 범위 설정 |

## 관측·운영 테이블
| 테이블 | 핵심 |
| --- | --- |
| `ai_sessions` | role(designer/writer/editor/chat/overseer…), model, status(`running|awaiting_user|done|error`), user_id, book_id, summary |
| `ai_steps` | session_id, idx, thought, skill, args(jsonb), observation(jsonb), prompt_snapshot, tokens_in/out, ms |
| `token_usage` | session_id, model, tokens_in/out, est_cost |
| `notifications` | target(user/role/broadcast), title, body, level, read_at |
| `app_settings` | key, value(jsonb), updated_by — 역할별 모델·rate limit·알림주기·안전강도 등 런타임 설정 |
| `audit_log` | admin_id, action, target, detail(jsonb) — 관리자 행위 감사 |

## RLS 핵심
- 헬퍼: `is_admin()`, `can_access_book(book_id)`(학생=본인, 교사=자기 학급, 관리자=전체).
- 학생은 자기 책/챕터/학습/편지만, 교사는 자기 학급 범위, 관리자는 전체.
- 관측 테이블 쓰기는 서비스 롤, 읽기는 관리자(일부 본인 범위).
- 게시판 `class_posts`: published는 같은 학급 멤버 읽기, 그 외는 작성 학생·담당 교사·관리자.

## 저장 원칙
- **거의 모든 데이터를 Supabase에 저장**(인메모리는 자동화 테스트 전용). 비테스트 환경에서 Supabase 자격이 없으면 기동을 거부한다(fail-closed).
- 삽화 등 파일은 Supabase Storage(`illustrations` public 버킷)에 올리고 공개 URL을 `chapters.illustration_path`에 둔다.
- 협업 문단은 `chapter_paragraphs`가 소스이고, `chapters.body`는 순서대로 재조립한 파생 캐시(독서·RAG·학습활동은 `body`를 읽어 하위호환).
