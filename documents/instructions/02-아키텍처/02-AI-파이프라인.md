# AI 파이프라인

> 일짜곰의 AI는 **Bible(단일 진실 원천)**을 중심으로 한 **4계층 역할** + **Skill 레지스트리/ReAct 오케스트레이터**로 동작하며, 모든 단계는 트레이스로 영속화되어 관리자 콘솔에서 추적된다.

## 1. Bible — 단일 진실 원천
책 한 권의 모든 정보(인물 카드·세계관·사건 시퀀스·교사 학습목표 분배·후반 큰줄기 `secretArc`)를 담은 구조화 JSON. 설계 AI가 책당 1회 생성하고, 집필·편집·삽화·대화 AI가 모두 이를 참조한다. 챕터가 늘어도 인물 성격·말투·삽화 외형이 일관된다. `bibles(book_id, data jsonb)`에 저장.

## 2. RAG (pgvector)
Bible과 본문 청크를 임베딩(`gemini-embedding-001`, 768차원)해 `chapter_chunks.embedding`(pgvector, HNSW cosine)에 저장한다. 집필 시 현재 챕터에 필요한 항목만 인출(`match_chunks(book_id, query, k)` RPC, 실패 시 Python 코사인 폴백)하므로 **30챕터 장편이 되어도 호출당 토큰이 일정**하다 — 학교 단위 비용 안정성의 핵심.

## 3. 4계층 역할 + 이미지
호출 빈도와 필요 품질이 반비례하는 지점에 모델 등급을 매칭한다.

| 계층 | 모델 | 빈도 | 역할 |
| --- | --- | --- | --- |
| 설계 | Gemini 2.5 Pro | 책당 1회 | Bible 생성, 학습목표를 챕터 사건으로 분배 |
| 집필 | Gemini 2.5 Flash | 챕터마다(SSE) | RAG 인출 → 본문 토큰 스트림 |
| 편집 | Gemini 2.5 Flash | 챕터 직후(비동기) | 말투·시간선·학년 어휘·학습목표 도달 검수, 미달 재생성, Bible 갱신 |
| 대화 | Gemini 2.5 Flash-Lite | 최빈 | 기획 인터뷰·튜터·인물 페르소나·유도모드 능동질문·자유집필 협업 |
| 이미지 | Imagen 4 | 챕터마다 1장 | Bible 인물 카드 입력 → 외형 일관 삽화 → Supabase Storage |

## 4. Skill 레지스트리
AI 능력을 `backend/app/ai/skills/`의 **스킬 1개 = 파일 1개**로 모듈화한다. 자동 수집되어 이름으로 호출되며, 모든 실행은 트레이스에 기록된다.

대표 스킬: `design_outline`(설계) · `generate_text`/`write_paragraph`(집필) · `retrieve_context`/`summarize`(RAG) · `update_bible` · `verify_objective`/`assess_flow`(검수·지도) · `generate_image` · `check_safety`/`assess_grade_vocab`(안전) · `ask_user`/`generate_choices`(대화) · `plan_interview`/`persona_reply`/`tutor_answer` · `generate_learning`/`ai_council`/`notify` · 총괄 AI 전용 `read_my_books`/`read_class_prompts`/`read_progress`/`read_my_activity`/`go_to_page`/`open_book`/`start_new_book`.

각 스킬은 `SkillContext`(store·gemini·settings·user·budget·`emit()`·`log_tokens()`)를 받아 실행하고 결과(observation)를 반환한다. 외부 효과 스킬은 실행 전 `check_safety`를 거친다.

## 5. ReAct 오케스트레이터
`backend/app/ai/react.py`는 **thought → action(skill) → observation** 루프로 목표를 달성한다.

```
start_session(role, goal, model)        # ai_sessions(status=running)
 └ loop (max_steps, budget):
     plan = decide(goal, [스킬 카탈로그], history)   # {thought, skill, args} | {finish}
     obs  = SKILLS[plan.skill].run(ctx, **args)
     emit(ai_steps: idx, thought, skill, args, observation, tokensIn/out, ms)
     # ask_user 스킬이면 status=awaiting_user 로 일시정지 → answer 엔드포인트로 재개
 └ end_session(status=done, summary)
```

ScriptedPlanner(결정적, 4계층 재구성·테스트용)와 LLMPlanner(실 Gemini JSON)를 둔다. `budget`·`max_steps`로 토큰·반복을 통제한다.

## 6. 자유집필 협업의 AI
기·승(free) 챕터는 통째 생성이 아니라 **학생 한 마디 = 한 문단 생성**이다.
- `chat.assess_flow` — 학생 의도가 (a) 직전 문단과 자연스러운가, (b) 학급 학습목표를 표현하는가 점검 → `generate`(생성) 또는 `coach`(지도) 결정. 지도는 "긍정 + 대안 + 근거" 형식, 결말은 절대 노출하지 않는다.
- `writer.write_paragraph` — 학생 의도 + 직전 문단 + RAG로 **한 문단**만 생성.
- `chat.next_paragraph_question` — 다음 진행 질문 1개. (자세히: `05-작동흐름/02-자유집필-협업.md`)

## 7. 총괄(Overseer) AI — 곰 작가
학생 메인 화면의 우측 사이드바 도우미. 학생 본인 데이터(내 책·진행률·학급 발제·활동)를 읽고, 화면 이동·새 책 생성 같은 액션을 돌려준다. `POST /ai/overseer/messages` → `{reply, actions[]}`. `role="overseer"` 세션으로 기록된다.

## 8. 트레이스 (관측)
모든 AI 동작이 영속화되어 "왜 이렇게 했는지" 추적 가능하다.
- `ai_sessions` — 작업 단위(role·model·status·user_id)
- `ai_steps` — ReAct 스텝(thought·skill·args·observation·tokens·**프롬프트 스냅샷**)
- `token_usage` — 호출별 토큰/비용
- `messages` — 사용자↔AI 대화(kind별)

관리자 콘솔은 이 데이터를 **사용자 → 책 → 단계 → 세션 → 스텝** 으로 드릴다운해 보여준다(→ `03-기능/03-관리자.md`).

## 9. 안전·비용
- 모든 입력/출력은 안전 게이트를 통과(부적절 차단, 정서 위험 보류 + `safety_flags`). Gemini/Imagen 콘텐츠 안전 설정 적용. 안전강도는 발제>학급>전역>코드 순으로 해석.
- 외부 AI 일시 오류(503/429)는 지수 백오프 재시도, 회복 불가 시 안전 폴백(placeholder/부분 결과)으로 강등.
- 모델 등급 분리 + RAG 고정 컨텍스트 + 사용자별 rate limit으로 비용 통제.
